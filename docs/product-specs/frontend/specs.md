# 前端高颜值客户端设计规范 (Frontend Specs)

本规格说明书定义了“自动化深度研究智能体”前端高颜值客户端的视觉设计标准、核心图表渲染规则、流式通信生命周期以及排版导出要求。前端实现时必须以此为依据。

---

## 1. 磨砂玻璃拟态视觉规范

前端必须采用现代暗黑磨砂玻璃拟态 (Dark Glassmorphism) 风格，营造极致的科技质感。

### 1.1 核心设计变量
在 CSS 样式表中，必须定义并严格使用以下 CSS 自定义属性：
*   **应用背景色 (`--bg-app`)**: 必须使用 `linear-gradient(135deg, #0f172a, #020617)`，呈现深靛蓝至极深灰黑的斜向渐变，以保障底层背景的沉浸感。
*   **玻璃背景色 (`--glass-bg`)**: 使用包含 45% 不透明度的深灰蓝色 `rgba(30, 41, 59, 0.45)`。
*   **玻璃高光边框 (`--glass-border`)**: 使用带有 8% 不透明度的纯白边线 `rgba(255, 255, 255, 0.08)`。在输入框聚焦或卡片高亮时，渐变至 `--color-primary`（科技蓝，`#38bdf8`）的不透明度 40% 版本。
*   **背景虚化滤镜 (`--glass-blur`)**: 规定 `backdrop-filter: blur(16px)`，保证玻璃背后的元素有足够的模糊度，避免文字干扰。
*   **阴影 (`--glass-shadow`)**: 使用深色扩散投影 `0 8px 32px 0 rgba(0, 0, 0, 0.37)` 增强卡片的悬浮悬空感。

### 1.2 微交互与动态效果
*   **过渡曲线**: 界面内所有尺寸、透明度、发光和颜色的渐变过渡均必须统一使用 `transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1)`（缓入缓出曲线），禁止生硬突变。
*   **悬浮发光**: 按钮及交互卡片在 hover 时，应通过 `box-shadow` 或伪元素渐显产生蓝绿色渐变发光（Glow Effect），光晕半径限制在 15px - 25px 之间。

## 2. 探索决策脑图 SVG 渲染规约

脑图展示 Planner Agent 发起的子查询探索路径，采用 SVG 进行轻量级动态拓扑图绘制。

### 2.1 节点层级拓扑计算
*   **坐标映射**:
    *   **根节点 (Level 0)**: 居中位于画布最左侧或顶部中心。
    *   **一阶子课题 (Level 1)**: 横向右移 $X_{diff}$（如 200px），并根据广度 $B$（Breadth）在 $Y$ 轴上均匀对称排开。
    *   **二阶深化查询 (Level 2)**: 在 Level 1 节点右侧进行再次分叉，纵向间距在 Level 1 的 $1/2$ 基础上收敛，避免发生节点碰撞。
*   **防碰撞边界**: 任意两个终点节点之间的垂直距离不得低于 40px。

### 2.2 节点状态色彩
圆圈节点 (`<circle>`) 必须通过不同的描边色与填充色标识其后台执行阶段：
1.  **未启动 (Pending)**: 填充 `rgba(148, 163, 184, 0.2)`，描边 `#64748b`。
2.  **搜索中 (Searching)**: 填充 `rgba(56, 189, 248, 0.2)`，描边 `#38bdf8`，并使用 `<animate>` 产生半径为 1.5 倍的呼吸扩散圆圈。
3.  **抓取中 (Scraping)**: 填充 `rgba(245, 158, 11, 0.2)`，描边 `#f59e0b`，带有旋转虚线边框效果。
4.  **已完成 (Completed)**: 填充 `rgba(52, 211, 153, 0.2)`，描边 `#34d399`。

## 3. 流式 SSE 状态管道机制

前端使用浏览器原生的 `EventSource` 连接后端 `/api/research/stream/{task_id}` 接口。

```
                    +--------------------+
                    |    Initializing    |
                    +---------+----------+
                              | EventSource 实例化
                              v
                    +--------------------+
         +--------->|     Connected      |<--------+
         |          +---------+----------+         |
         |                    |                    |
         | 收到 log/          | 收到 state_update   | 15秒心跳
         | 增加日志流          | 脑图/计数指标刷新   | (: heartbeat)
         |                    |                    |
         |                    v                    |
         +--------------------+--------------------+
                              |
                     +--------+--------+
                     |                 | 发生异常/连接中断
                     v                 v
            +----------------+  +--------------+
            |    Complete    |  |    Error     |
            +-------+--------+  +------+-------+
                    |                  |
                    +--------+---------+
                             | 自动/手动调用 close()
                             v
                    +--------------------+
                    |       Closed       |
                    +--------------------+
```

### 3.1 事件路由规约
客户端接收到的 SSE 数据帧必须按照 `event` 类型字段分发至对应的数据处理器：
*   **`event: log`**: 将 `data.message` 及格式化后的时间戳插入全局日志数组，触发控制台自动滚动。
*   **`event: state_update`**: 更新前端的 `current_depth`、`sources_count` 与 `facts_count` 指标，同时驱动脑图绘制更新。
*   **`event: complete`**: 提取 `data.report`，将其注入 Markdown 渲染器，关闭当前 EventSource 通道，将系统置为就绪态。
*   **`event: error`**: 气泡提示错误信息，将错误消息写入控制台末尾，执行资源释放。

### 3.2 异常容错与熔断要求
*   **心跳超时判定**: 设定 25 秒定时检测器。如果在 25 秒内未收到后端发送的任何事件或心跳注释 (`: heartbeat`)，判定连接已死，执行 `close()` 并发起重试。
*   **意外中断处理**: 若发生 `EventSource.onerror` 且任务并非主动完成，限制最大自动重连次数为 3 次，超过 3 次则向用户弹出断网警示，并将任务置为 `failed` 态。

## 4. Markdown 与学术公式排版

渲染引擎负责将 Synthesizer Agent 输出的 Markdown 报告编译为结构严密的 HTML 页面。

### 4.1 LaTeX 公式转译规则
*   **行内公式**: 匹配被单个 `$` 包裹的文本（如 `$f(x) = \sin(x)$`），调用 KaTeX 的行内渲染模式，垂直对齐至基线。
*   **独立块公式**: 匹配被双 `$$` 包裹的公式段落（如 `$$\nabla \times \mathbf{E} = -\frac{\partial \mathbf{B}}{\partial t}$$`），渲染为居中显示的数学块，若溢出则支持横向滚动。

### 4.2 引用文献 Popover 卡片
*   **DOM 匹配**: 编译 Markdown 时，将匹配到的 `[1]` 或 `[2]` 渲染为带特有 `class="ref-anchor"` 的 `<a>` 标签。
*   **Popover 交互**:
    *   鼠标悬浮于引用锚点时，使用 absolute 定位计算弹出框位置。
    *   弹出框同样采用 Glassmorphism 样式，毛玻璃虚化度 `--glass-blur`，内呈银白色字体，包含引用的 URL、文献标题以及后端提供的切片事实（evidence）。
    *   移除鼠标时，设置 300ms 延时销毁以保证用户操作体验。

### 4.3 A4 打印与 PDF 导出排版重写
当用户点击“打印 PDF”时，前端必须通过 `@media print` 覆写全局 CSS 属性，确保纸张打印效果：
*   **版面隐藏**: 隐藏侧边配置面板、日志控制台、顶栏导航及所有悬浮控制按钮。
*   **纸张边缘适配**: 将正文宽度设为 `100%`，去除背景渐变、虚化属性，将背景统一设定为 `white`，文字颜色设定为极深灰色 `#1e293b`。
*   **防分页中断**: 为所有标题和代码块加上防分页截断声明：
    ```css
    h1, h2, h3 { page-break-after: avoid; }
    pre, blockquote { page-break-inside: avoid; }
    ```
